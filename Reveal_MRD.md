# Reveal — 市場需求文件 (MRD)

## 一、產品概述

### 產品名稱
Reveal（暫定）

### 一句話定位
一鍵捕捉此刻的空氣感，把照片變成一封活的信。

### 產品本質
Reveal 是一台「氛圍捕捉器」。利用手機感測器與環境數據，將任何一張照片轉化為一個帶有聲音、動態視覺、手寫筆跡的沉浸式體驗物件。不是濾鏡，不是特效，而是讓收件人花 30 秒短暫地走進寄件人當時身處的世界。

### 核心價值主張
- 照片傳遞的是「我看到了什麼」
- Reveal 傳遞的是「我當時身處什麼樣的世界裡」
- 保留寫明信片的心意和儀式感，去掉物流的麻煩，加上實體做不到的氛圍封裝

## 二、市場分析

### 目標市場
- 數位禮品市場（全球約 $260 億美金，年增 10%+）
- 旅行紀念品市場
- 婚禮/活動體驗市場
- 品牌體驗行銷市場

### 目標用戶畫像
**主要用戶：** 20-35 歲，重視生活美感，使用 Dazz/VSCO 等工具拍照，願意為情感表達花時間和金錢的人

**使用場景：**
- 旅行中的風景瞬間
- 演唱會、音樂祭
- 小孩的成長片刻
- 一個人的安靜時刻
- 雨天的咖啡廳
- 生日、紀念日、節日送禮
- 任何想記住「此刻空氣感」的瞬間

### 競品分析

| 競品 | 它做的事 | Reveal 的差異 |
|------|---------|--------------|
| Instagram | 照片分享 | 3 秒被滑過，沒有沉浸感 |
| Dazz/VSCO | 照片濾鏡 | 只有視覺，沒有聲音和互動 |
| Polarsteps | 旅行軌跡記錄 | 記錄工具，不是體驗物件 |
| TouchNote | 手機寄實體明信片 | 只有實體，沒有數位體驗 |
| Slowly | 數位信件 | 純文字，沒有氛圍封裝 |
| 抖音/TikTok 濾鏡 | 動態特效 | 即用即棄，不是可保存的物件 |

**Reveal 的獨特位置：** 市面上沒有產品在做「把環境氛圍封裝成可體驗物件」這件事。

## 三、產品設計

### 核心流程

#### 階段一：捕捉（當下 15 秒）
用戶按下捕捉按鈕，系統在背景自動完成：
- 錄製 15 秒環境音
- 讀取 GPS 座標 + 海拔
- 讀取加速度計（靜止/步行/移動中）
- 讀取麥克風音量波動（判斷孤獨/人群密度）
- 拍一張照片（或稍後從相簿選）
- 打天氣 API 取得溫度、濕度、天氣狀態、風速
- 從照片分析環境光色溫

用戶感知到的只有一件事：按下按鈕，等 15 秒。

**替代路徑（降低門檻）：**
用戶也可以直接從相簿選一張已經拍好的照片，系統透過 EXIF 數據（時間 + GPS）回溯查詢天氣 API 取得歷史環境數據。環境音可選擇事後補錄，或由 AI 根據場景數據自動生成。

#### 階段二：製作（事後 3 分鐘）
1. 選擇/更換照片（從相簿或當時拍的）
2. 手寫一段話（手指在 Canvas 上書寫，記錄座標 + 時間戳 + 壓力）
3. 也可選擇打字（備案，UI 上不突顯）
4. 系統自動生成氛圍視覺 + 環境音景 + 數位郵票
5. 預覽完整的動態明信片
6. 選擇：留給自己 / 產生分享連結送給別人 / 印成實體寄出

#### 階段三：體驗（收件人 / 未來的自己）
1. 打開連結，畫面全黑或全模糊
2. 長按螢幕，內容開始「顯影」— 如同暗房沖洗底片（5-10 秒）
3. 環境音景自動播放
4. 照片帶有氛圍紋理層動態效果
5. 手寫文字按照原始速度、停頓、力道一筆一畫重播
6. 角落顯示自動生成的數位郵票
7. 底部顯示捕捉地點、日期、環境數據摘要

### 核心功能詳述

#### 1. 氛圍生成系統

照片只是原料。系統根據環境數據，把照片「染」上那個瞬間的空氣感。

**方法：** 先準備一組 AI 生成的美術基底模板（20-30 張），系統根據環境數據自動匹配，再用數據驅動的參數調變模板。

**輸入數據 → 視覺/聽覺參數映射：**

| 感測器數據 | 控制的視覺參數 |
|-----------|--------------|
| 照片平均色調 | 整張明信片的基礎色溫 |
| 溫度 + 濕度 | 畫面質感（乾冷=銳利清透、濕熱=霧氣朦朧、悶熱=邊緣熱浪扭曲） |
| 音頻低頻佔比 | 動畫呼吸速度（低頻重=緩慢沉穩） |
| 音頻高頻佔比 | 光點/粒子閃爍密度 |
| 靜止/步行/移動 | 所有動畫的整體速度與方向性 |
| 音量波動方差 | 元素疏密（安靜=大量留白、吵雜=密集） |
| 天氣狀態 | 疊加層效果（雨=水滴漣漪、晴=光斑、陰=灰調壓低） |
| 風速 | 粒子飄動幅度 |

**自動個性化（不是用戶選，是環境決定）：**
- 山頂 → 風聲 + 銳利清透 + 緩慢呼吸
- 海邊 → 浪聲 + 潮汐動畫 + 波浪節奏
- 夜市 → 人聲 + 暖色光暈 + 密集光點
- 雨中 → 雨聲 + 霧氣朦朧 + 水滴漣漪

**下限保障：** 無論數據組合如何，至少都是一張設計師等級的美圖加了個性化。上限有驚喜：極端環境數據會產出特別獨特的變體。

#### 2. 手寫筆跡系統

使用 Canvas API 捕捉手指觸控軌跡：
- `touchstart` → 筆落下
- `touchmove` → 筆移動，記錄座標 (x, y) + 時間戳 + 壓力 (touch.force)
- `touchend` → 筆抬起

**重播：** 按照原始時間戳重現書寫過程。寫得快的地方重播也快，猶豫停頓的地方也會停頓。收件人看到的是一筆一畫慢慢出現，不是靜態字。

**美化處理：** 貝茲曲線平滑（去除鋸齒但保留真實的歪扭），不做字體模擬。潦草本身就是人味。

#### 3. 環境音景系統

**有實錄音：** 用戶錄的 15 秒原始音訊，經由 Web Audio API 做：
- 高通濾波降低風切雜音
- 爆音自動壓縮
- 循環處理讓 15 秒可以無縫重複播放
- FFT 頻譜分析拆出低/中/高頻比例，驅動視覺參數

**無實錄音：** 根據場景數據（地點類型 + 天氣 + 時間）用 AI 生成擬真環境音景。標注「AI 生成音景」。

#### 4. 顯影互動

收件人打開連結時不會直接看到內容：
- 初始狀態：`filter: blur(30px)` + 黑色遮罩
- 長按螢幕：blur 值從 30 漸減到 0，遮罩透明度漸減
- 手放開：暫停在當前狀態
- 約 5-10 秒完成全部顯影
- 模擬拍立得 / 暗房沖洗底片的體驗

**設計目的：** 製造期待感和揭曉的興奮，讓「打開」這件事本身就是體驗的一部分。

#### 5. 數位郵票系統

每張明信片自動生成一枚獨一無二的數位郵票。

**生成邏輯：**
- GPS 座標數字作為亂數種子
- 溫度決定色調
- 濕度決定線條模糊程度
- 海拔決定圖案複雜度
- 天氣狀態決定基本形狀（圓形、波浪、稜角等）

**商業用途：**
- 郵票是付費的載體 —「買一張郵票寄出」比「付費解鎖」心理負擔小
- 可推出限定版郵票（節日限定、城市限定、季節限定）
- 天然的收藏品系統

#### 6. 實體明信片輸出

用戶做完數位版後可選擇印成實體卡片寄出：
- 正面：照片 + 氛圍視覺 + 手寫字跡 + 郵票圖騰（合成為靜態圖）
- 背面：QR code，掃描後進入完整動態體驗
- 透過隨需印刷 (Print on Demand) API（如 Lob、Prodigi）印刷寄送
- 一張卡片同時是實體紀念品和數位體驗入口

---

### 設計原則

1. **保留摩擦力** — 製作過程刻意需要 1-3 分鐘，不能一鍵完成。投入的時間就是心意的證明。
2. **環境決定個性** — 不給用戶一堆開關自己選，讓那個地方、那個瞬間自己長出明信片的樣子。
3. **體驗 > 資訊** — 收件人花 30 秒沉浸在一個瞬間裡，而不是 0.5 秒看完一張圖。
4. **人的痕跡** — 手寫字跡、親自錄的環境音，這些不完美但真實的東西才是溫度來源。

## 四、技術架構

### 前端（核心）
- **框架：** React（PWA，不做原生 App，零下載門檻）
- **手寫：** Canvas API + Touch Events
- **音訊：** Web Audio API（錄製、FFT 頻譜分析、音景播放、濾波處理）
- **視覺效果：** CSS filters + blend modes + Canvas 2D 動畫（MVP 不用 WebGL）
- **感測器：** Geolocation API、DeviceMotion API、MediaRecorder API、Ambient Light Sensor
- **顯影互動：** CSS transition + touch event 控制進度

### 後端（最小化）
- **存儲：** Cloudflare R2 或 AWS S3（音檔 + 圖片 + 軌跡數據）
- **連結服務：** 短連結生成 + 靜態頁面 hosting
- **API Proxy：** 天氣數據代理（避免前端暴露 API Key）
- **資料庫：** Supabase 或 PlanetScale（用戶帳號 + 明信片 metadata）

### 第三方服務
- **天氣：** OpenWeather API（免費方案 60 次/分鐘，支援歷史數據查詢）
- **印刷：** Lob API 或 Prodigi API（隨需印刷 + 全球寄送）
- **AI 音訊：** 環境音景生成（無實錄時的替代方案）
- **AI 圖片：** Midjourney / DALL-E（一次性生產模板素材，非 runtime 呼叫）

### 關鍵技術風險與對策

| 風險 | 對策 |
|------|------|
| iOS Safari Web Audio 限制（需用戶互動才能播放） | 顯影的長按互動本身就是用戶互動事件，在此觸發音訊播放 |
| iOS DeviceMotion 需要授權 | 首次使用時一個授權彈窗 |
| 錄音品質差（風切、爆音） | 高通濾波降風切，爆音自動壓縮 |
| PWA 背景錄音在 iOS 上不穩定 | 限制為前台操作，15 秒內完成 |
| 載入速度 | 音檔壓縮為 opus/webm，圖片壓縮為 webp，目標 3 秒內載完 |
| 某些數據組合產出的視覺不好看 | 基底模板保底，數據只做調變不做生成 |
| 手寫字鋸齒感 | 貝茲曲線平滑處理 |

## 五、商業模式

### 收入來源

#### 1. 數位明信片（C2C）
- **定價：** $2 美金/張（約 60 台幣）
- **免費版：** 可製作但有浮水印 + 體驗限時 7 天
- **付費版：** 完整體驗 + 永久保存 + 筆跡重播
- **付費行為包裝：** 「買一張郵票寄出」而非「付費解鎖」

#### 2. 實體明信片印刷寄送
- **定價：** $8 美金/張（約 240 台幣）
- **成本：** $3 美金/張（印刷 + 寄送）
- **毛利：** $5 美金/張
- 正面實體 + 背面 QR code 連結動態版

#### 3. 限定郵票
- **定價：** $1-2 美金/張
- **類型：** 節日限定、城市限定、季節限定
- **成本：** 零（純數位商品）
- **作用：** 持續性收入 + 收藏驅動回訪

#### 4. 婚禮/活動套餐（B2C 高客單）
- **定價：** $1,000-1,300 美金/場
- **內容：** 婚禮當天氛圍封裝 + 實體卡片作為婚禮小物 100 張 + 客製化服務
- **目標：** 10 場/月 = $10,000-13,000 月收入

#### 5. 品牌體驗行銷（B2B）
- **客戶：** 精品飯店、咖啡廳、展覽、音樂祭
- **定價：** $200-3,000 美金/月/間
- **價值：** 「帶走你在這裡的體驗」— 品牌差異化工具
- **目標：** 10 個客戶 = $5,000-15,000 月收入

#### 6. 追思/紀念（B2C 高客單）
- **定價：** $50-200 美金/次
- **內容：** 客製化封裝過世親人的照片 + 地點環境音 + 手寫字跡掃描
- **成本：** 幾乎為零

#### 7. 情侶訂閱「氛圍信箱」
- **定價：** $5 美金/月/對
- **內容：** 每週互相寄一封氛圍明信片，固定的相處儀式
- **留存率預期高於一般用戶**
- **目標：** 1,000 對 = $5,000 月收入

#### 8. API / SDK（遠期）
- **定價：** $0.05-0.10 美金/次呼叫
- **客戶：** 日記 App、旅行 App、社交 App
- **前提：** 核心技術成熟後才走這條路

### 收入結構預估（每 1,000 用戶）

| 行為 | 比例 | 人數 | 花費 | 收入 |
|------|------|------|------|------|
| 打開看看就走了 | 60% | 600 | $0 | $0 |
| 做了免費版，沒付費 | 25% | 250 | $0 | $0 |
| 付費買數位版 | 10% | 100 | $2 | $200 |
| 加購實體印刷 | 4% | 40 | $2+$8 | $400 |
| 加買限定郵票 | 1% | 10 | $12 | $120 |
| **合計** | | | | **$720** |

每用戶平均貢獻：$0.58 美金（約 17 台幣）

**注：** 以上僅計算 C2C 部分。B2B、婚禮、追思等高客單場景另計。

### 商業策略
- **C2C 明信片養品牌**（讓產品被看見、被分享）
- **B2B 和婚禮養現金流**（支撐營運和生活費）
- 兩條線同時進行，不等 C2C 賺錢才做 B2B

## 六、分享與增長策略

### 分享機制
- 不做 App 內社群，不做 feed
- 產出獨立網頁連結，任何裝置瀏覽器打開即可體驗
- 收件人不需要註冊或下載
- 透過 LINE、IG story、訊息等現有社交平台自然分享

### 增長飛輪
1. 寄件人做一張明信片送出
2. 收件人打開，被體驗震撼
3. 收件人截圖/錄影分享到社群「你看他送我這個」
4. 收件人的朋友看到，想自己也做一張
5. 新用戶進入 → 回到第 1 步

**核心驗證指標：** 收件人 → 寄件人的轉換率。如果 > 10%，飛輪成立。

### 免費版策略
讓免費版就能體驗到 80% 的震撼（有顯影、有音景、有手寫），付費買的是完整度（永久保存、高品質、無浮水印）。收到的人必須被打動，才會變成下一個寄件人。

## 七、發展路線圖（保守估計，含 AI 輔助開發）

### 階段 0：開發 MVP（第 1-5 週）

**交付物：**
- PWA 網頁應用
- 照片匯入 + 天氣 API 回溯
- 手寫 Canvas + 筆跡重播
- 3-5 個氛圍模板 + 基本數據驅動視覺
- 環境音錄製 + 播放
- 顯影互動
- 分享連結
- 郵票自動生成

**成本：** 域名 + hosting，約 $10-20/月

### 階段 1：驗證（第 2-4 個月）

**目標：** 驗證收件人會不會變成寄件人

**動作：**
- 發 PTT、Dcard、Twitter、Reddit、Product Hunt
- 拍示範影片投社群
- 找 50-100 人試用

**預期數據：**
- 累計用戶：200-800 人
- 付費用戶：5-15 人
- 月收入：$10-30 美金

### 階段 2：找增長（第 4-10 個月）

**目標：** 找到可重複的增長引擎

**動作：**
- 加入實體印刷功能
- 節日行銷（情人節、母親節、聖誕節）
- 小額廣告測試（$50-100/月）
- 開始接觸婚禮市場和 B2B

**預期數據：**
- 累計用戶：2,000-8,000
- 月收入：$180-540 美金（C2C）+ B2B 另計
- 月利潤：$80-290 美金

### 階段 3：能不能活（第 10-18 個月）

**目標：** 月收入接近生活費

**動作：**
- 英文版上線
- 穩定 B2B 客戶 5-10 間
- 婚禮市場 3-5 場/月
- 限定郵票系列上線

**預期數據（保守）：**
- 累計用戶：10,000-50,000
- C2C 月收入：$400-3,000
- B2B + 婚禮月收入：$2,000-8,000
- 總月利潤：$1,500-6,000 美金

### 階段 4：規模化（第 18-36 個月）

**目標：** 穩定且可增長的事業

**動作：**
- 多語言版本（日文、韓文、西班牙文）
- 旅遊平台合作
- 情侶訂閱制上線
- 可能需要請 1-2 人

**預期數據：**
- 累計用戶：10-30 萬

## 十、關鍵成功指標 (KPIs)

### 階段 1 必須驗證
1. **完成率：** 開始製作的人中，多少比例完成並送出？目標 > 40%
2. **收件人轉換率：** 收到明信片的人中，多少比例自己也做了一張？目標 > 10%
3. **分享率：** 收件人截圖/錄影分享到社群的比例？目標 > 5%

### 長期追蹤
4. **付費轉換率：** 使用者中付費的比例。目標 > 8%
5. **每用戶平均收入 (ARPU)：** 目標 > $0.50
6. **月活躍用戶 (MAU)：** 持續增長
7. **B2B 客戶數：** 穩定增長

## 十一、風險總覽

| 風險 | 嚴重程度 | 對策 |
|------|---------|------|
| 收件人不會變成寄件人（無自然增長） | 致命 | 階段 1 優先驗證，不成立則調整產品 |
| 旅途中用戶不願意花時間操作 | 高 | 相簿匯入路徑讓「當下」零操作 |
| 低頻使用，用完就走 | 高 | 擴大場景（不限旅行）+ B2B 補收入 |
| 免費替代品能達到 80% | 中 | 自動化氛圍封裝是免費工具做不到的 |
| iOS 瀏覽器相容性 | 中 | 針對 Safari 逐一測試和處理 |
| 一人團隊精力不足 | 中 | AI 輔助開發 + MVP 範圍嚴格控制 |
| 視覺品質不穩定 | 中 | 基底模板保底 + 持續調參迭代 |
| 存儲成本隨用戶增長 | 低 | 壓縮策略 + 免費版限時存儲 |
