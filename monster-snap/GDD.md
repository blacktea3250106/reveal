# MonsterSnap — 遊戲設計文件 (GDD)

## 一、產品概述

### 遊戲名稱
MonsterSnap（暫定）

### 一句話定位
拍照捕捉真實世界的物體，AI 將它們變成獨一無二的怪獸，養成、進化、對戰。

### 核心玩法循環
**拍照 → 獲得怪獸 → 養成 → 對戰 → 拍更多照片**

---

## 二、核心系統

### 1. 拍照捕捉系統

**流程：**
1. 玩家打開相機拍攝任何物體（貓、花、車、食物、雲...）
2. 照片送至 AI 辨識（Google Cloud Vision / OpenAI Vision）
3. AI 回傳辨識結果：物體類別、顏色、環境
4. 系統根據辨識結果生成一隻怪獸

**AI 辨識 → 怪獸映射規則：**

| 辨識結果 | 怪獸屬性 | 範例 |
|---------|---------|------|
| 動物（貓、狗、鳥） | 野獸系 | 拍到貓 → 影爪貓 |
| 植物（花、樹、草） | 草木系 | 拍到玫瑰 → 荊棘薔薇獸 |
| 火/光源（蠟燭、燈、太陽） | 火焰系 | 拍到蠟燭 → 燭幽靈 |
| 水/液體（海、河、飲料） | 水流系 | 拍到海 → 深淵海龍 |
| 金屬/機械（車、電腦、工具） | 機甲系 | 拍到車 → 鋼鐵衝撞獸 |
| 食物 | 美食系 | 拍到拉麵 → 沸騰麵龍 |
| 天空/雲/天氣 | 風暴系 | 拍到烏雲 → 雷暴鷹 |
| 岩石/土地/建築 | 岩地系 | 拍到石頭 → 巨岩守衛 |
| 幽暗/夜景/陰影 | 暗影系 | 拍到夜景 → 暗夜魅影 |
| 其他/難辨識 | 混沌系 | 隨機稀有怪獸 |

**怪獸獨特性：**
- 每隻怪獸的基礎數值根據照片的 AI 分析結果隨機浮動（±15%）
- 照片的主色調影響怪獸外觀配色
- 拍攝時間（日/夜）影響怪獸變體
- 同一類物體可重複拍，但每隻數值不同（像寶可夢的 IV 值）

**稀有度系統：**
- 普通（Common）：60% — 常見物體
- 稀有（Rare）：25% — 不常見組合或較罕見物體
- 史詩（Epic）：12% — 特殊條件觸發（如：特定時間、天氣、地點）
- 傳說（Legendary）：3% — 極端條件（如：拍到彩虹、極光、罕見動物）

### 2. 怪獸養成系統

**基礎屬性：**
- HP（生命值）
- ATK（攻擊力）
- DEF（防禦力）
- SPD（速度）— 決定出手順序
- 屬性（系別）— 決定相剋關係

**等級系統：**
- 最高 Lv.50
- 經驗值來源：對戰勝利、餵食、每日任務
- 每升 5 級可學一個新技能（從技能池中選）

**進化系統：**
- Lv.15 → 第一階段進化
- Lv.35 → 第二階段進化（最終型態）
- 進化改變外觀、大幅提升數值、解鎖終極技能

**餵食系統：**
- 拍攝食物類照片 → 產生「飼料」道具
- 不同食物給不同屬性加成
- 每日餵食上限 3 次（防止刷太快）

### 3. 回合制對戰系統

**屬性相剋表：**

```
火焰 > 草木 > 水流 > 火焰（基本三角）
機甲 > 風暴 > 岩地 > 機甲（進階三角）
暗影 <> 美食（互相剋制）
野獸：無特別弱點，基礎數值較高
混沌：隨機觸發任意屬性效果
```

**對戰規則：**
- 每位玩家攜帶 3 隻怪獸上場
- 回合制，速度快的先出手
- 每隻怪獸有 4 個技能欄位
- 屬性相剋造成 1.5x 傷害，被剋 0.75x
- 可中途換獸（消耗一回合）

**技能類型：**
- 攻擊技能（物理/特殊）
- 防禦技能（減傷/反彈）
- 增益技能（加攻/加速/回血）
- 狀態技能（中毒/燒傷/麻痺/凍結）

**對戰模式：**
- **PvE 冒險模式：** 章節制關卡，打野生怪獸和 Boss
- **PvP 排位賽：** 即時配對，ELO 積分制
- **好友對戰：** 分享房間碼

### 4. 圖鑑系統

- 按系別分類，收集所有怪獸
- 顯示已捕捉/未捕捉（剪影提示）
- 每隻怪獸頁面保留原始捕捉照片
- 收集里程碑獎勵（集滿一個系別 → 獎勵）

## 三、技術架構

### 技術棧選擇：React Native + Expo

**為什麼選 React Native：**
- 一套 code 同時出 iOS + Android
- 相機、檔案系統等原生功能透過 Expo 輕鬆存取
- JavaScript/TypeScript 生態豐富
- 對獨立開發者最友善

### 前端
- **框架：** React Native + Expo (Managed Workflow)
- **狀態管理：** Zustand（輕量、簡單）
- **導航：** Expo Router (file-based routing)
- **相機：** expo-camera
- **動畫：** React Native Reanimated + Lottie（怪獸動畫）
- **對戰畫面：** React Native Skia（2D 繪圖引擎）

### 後端
- **框架：** Node.js + Hono（輕量 API）
- **資料庫：** Supabase（PostgreSQL + Auth + Realtime）
- **即時對戰：** Supabase Realtime 或 Socket.io
- **圖片存儲：** Supabase Storage 或 Cloudflare R2
- **部署：** Cloudflare Workers 或 Railway

### AI 服務
- **圖片辨識：** OpenAI Vision API（gpt-4o-mini，便宜且準確）
- **怪獸圖片生成：** 預先用 AI 生成怪獸素材庫（不做 runtime 生成，太慢太貴）
- **備案：** Google Cloud Vision API

### 怪獸素材策略
- MVP 先準備 **50 隻基礎怪獸**（10 個屬性 × 5 隻）
- 每隻有 3 階段進化圖 = **150 張怪獸圖**
- 用 AI 批次生成（Midjourney/Stable Diffusion），統一風格
- 照片辨識結果 → 映射到最匹配的怪獸模板 + 色調調整

## 五、畫面流程

```
[首頁]
  ├── [拍照捕捉] → 相機 → AI 辨識中... → 獲得怪獸！→ 怪獸詳情
  ├── [我的怪獸] → 怪獸列表 → 怪獸詳情（升級/進化/技能）
  ├── [對戰]
  │     ├── [冒險模式] → 選關卡 → 選隊伍(3隻) → 對戰畫面
  │     └── [PvP] (後續版本)
  └── [圖鑑] → 按屬性分類瀏覽
```
